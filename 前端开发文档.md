# 1. 本地环境部署
## 1.1 开发环境配置
开发环境是 `node 24.13.0`， 本地部署前请先确认是否安装 `node 24.13.0`。推荐使用 `nvm` 工具管理 `node` 版本。
## 1.2 项目路径
项目在运行 `npm install` 前先确保处于正确的运行路径下，路径应该是 `homepage/frontend`。
## 1.3 项目提交
本项目采用 Github 自动打包页面，克隆项目后提交到仓库会自从触发部署流程，每次提交后请检查是否部署成功。
# 2. 项目和页面规范
# 2.1 页面结构
本项目使用 `Tailwind + vue3 + Nuxt UI v4.5 + TypeScript` 进行开发，请在编写前查阅官方文档，来实现功能。多语言模块使用 `i18n` 插件。在项目开发中请避免使用硬编码的方式开发，以便于后期维护。
## 2.2 AI生成提示词
### 如果你想要使用 AI 来开发页面，请预先使用以下提示词：
请为我生成一个页面。要求遵循“社论学术风”规范：


数据驱动：在 `script setup` 中定义 `interface` 和数据数组。

i18n 隔离：组件内只保留 i18n 键值路径，不准出现字面量。

布局细节：使用 Tailwind 的语义化 Token 确保支持深色模式。

组件化：优先使用 Nuxt UI 组件，并增加平滑的入场动效。
## 2.3 项目文件存放
目前所有的前端文件均放在路径 `frontend/src/modules/showcase` 目录下，推荐新增项目文件时按照其功能或者路由分类命名存放。

# 3 页面开发
## 3.1 新增页面
当你想要新增一个页面时，请确保其出现在 `frontend/src/modules/showcase/router/index.ts` 中,如有放在导航栏的必要，请检查 `frontend/src/modules/showcase/components/TheHeader/TheHeader.vue` 中。这可能会需要一点 Vue3 的基础知识。
